Bank Data File Structure

 

The file to be imported by the Import Bank Data utility needs to meet certain criteria in order to be imported properly.

 

This needs to be a text file, usually provided by the bank. The text file will contain information for cleared checks that will automatically update the check items on an existing bank reconciliation. The bank text file will need to be named BR3 with any extension, and will need to be placed in the data directory on the DB Distributor server.

 

The 80 character file layout needs to be in the following format:

 

Field Starting Position, Length
Blank

1,3
Account Number 4,10
Check Number 14,10
Type Code 24,1
Issue Date 25,6 (MMDDYY)
Check Amount 31,10 ($7,859.36 = 0000785936)
Blank 41,15
Check Paid Date 56,6 (MMDDYY)
Blank 62,3
Account Number 65,10
Unused 75,6

 

At this time, this is the only format available for import. It is known as format "01" or the "Generic Master".

 

The logic used when importing the file is as follows:

 

If a matching check entry in the bank rec is found, the check amounts will be compared. If they match, the check will be cleared. If the amount is different, the bank amount will be updated to the bank rec detail, but will be flagged as a discrepancy.

 

A log file (BR_LOG) will be written to during the process. This log file will be amended each time the process is run. The following information will be recorded in the log file for each BR3 file processed:

 

Date/Time user, total records to process

Details of any record that has a problem

Number of records cleared with exact amounts

Number of records cleared with discrepancies

Number of records not found in the bank rec

Number of records not processed

Total number of records processed

 

After the BR3 file is processed, it will automatically be renamed BR4, plus the same extension, for archive purposes. When the update is complete, and the log file is checked, the user will then return to the bank rec data entry.
