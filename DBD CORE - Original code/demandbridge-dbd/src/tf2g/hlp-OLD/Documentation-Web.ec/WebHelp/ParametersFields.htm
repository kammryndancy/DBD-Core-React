EC Parameters dialog fields and settings

The EC Parameters dialog is used to specify the system-wide operating parameters for Web.ec. The EC Parameters dialog is divided into four tabs. Click on any of the links below to see the field descriptions and the setting options and requirements for each tab. Click on the "Back To Top" link associated with each table to return to these links.

Image path

The Image path field indicates where the web page images (i.e., gif and/or jpeg files) are located.

Note: This needs only to be set if the images are not being stored in separate directories by customer, which is the recommended method.

Note: If the images are being kept on a server other than your web server, a complete URL must be specified which points to the location of the images (see example below).

For example: If the images for a customer named "ABC" reside on a remote server hosted by an independent ISP with an IP address of "166.102.29.126", then the complete URL for this field would be: "http://166.102.29.126/images/ABC/". If only a particular image from this directory is needed, the path would also require adding the tag "?tf?image_form1?" at the end of the URL. For example: To display an individual image of a Custom Item for this same customer named "ABC", the path might be something like: "http://166.102.29.126/images/abc/?tf?image_form1?".

Valid path to the image directory

Limit usage reporting on PC detail to active Ship-to?

The ‘active shipto’ is the current ship-to that is assigned to the in-process webec order being built by the buyer. This option is only effective if your web site prompts the user for the ship-to location prior to the display of the product catalog.

Option Comments

N = The usage on the product catalog will be driven based on the location setting on the buyer parameters record. If the location field is blank then webec will report usage for all locations for the associated customer and item.

Y = The usage on the product catalog will be driven based on the currently active ship-to selected by the buyer. For location based buyers this will be the same as the location selection on their buyer parameters record (since they have no option to change this value). For corporate purchasing agents, this will allow you to limit the reported usage for the selected product based solely on the ship-to they have currently selected during their on-line ordering session.

Y / N

Change 'attention' with every change in Ship-to?

This field controls the resetting of the attention field if a new ship-to is selected from the ship-to drop down list.

Option Comments

N = Once the attention field is defaulted to a non-blank value, it will not change with subsequent changes to the ship-to

Y = With each change to the selected ship-to location, using the ship-to drop down list, webec will change the defaulted attention field to match the attention field on the newly selected ship-to record.

Note: This option is disabled if the current buyer has a default attention value set on their buyer parameters.Note:  If the selected ship-to record does not have an attention field value, then no change will be effected to the current on-line attention field value.

Y / N

Activate order/approval functions?

This checkbox allows you to restrict the ship-via drop down list for all buyers associated with this customer. If checked, the ship-via drop down list for all buyers associated with this customer will contain only the default ship-via method that is associated with the selected TopForm ship-to record. This feature requires that the default ship-via method be set for all ship-to’s of the Web.ec customer. If unchecked, the ship-via drop down list is not restricted.
