Defining New Web.ec Sort Codes

This procedure should only be completed by personnel familiar with sort code definitions. If you incorrectly define your new sort code, your Web.ec system will not function properly.

IMPORTANT NOTE: TopForm pre-defines a number of sort codes for your use (DDL_SHIPTO, EMAILSUB_O, OS_BYDATE, etc.). You can use the UD Sort Element Definition item to define the data and text elements used in these sort codes, but do not modify these codes.

To define a new Sort Code, complete the following actions.

  1. Launch the Web.ec module.

  2. Select the "UD Sort Codes" item from the User Defined menu. The User Defined Sort Code Maintenance dialog appears with all fields empty.

  3. Enter the new Sort Code name.

  4. Define the parameters for the new sort code according to the requirements and options specified in the Dialog Fields and Settings for this topic.

  5. Click the button to commit the field entries and configure the Web.ec sort code settings.

  6. Click the button to close the User Defined Sort Code Maintenance dialog.

  7. Define the text and/or data elements that make up the new Sort Code using the UD Sort Element Definition item.

STOP. You have completed this task.
