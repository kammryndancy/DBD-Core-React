EC Customer Parameters dialog fields and settings

The EC Customer Parameters dialog is used to specify the system-wide operating parameters for a web customer. The EC Customer Parameters dialog is divided into seven tabs. Click on any of the links below to see the field descriptions and the setting options and requirements for each tab. Click on the "Back To Top" link associated with each table to return to these links.

1. Images tab

Back To Top

Field Name

Purpose / Use

Setting Options and Requirements

Customer Code

Enter or select the Customer Code from the search list. Customer codes are set using the TopForm Accounts Receivable module.

Valid TopForm Customer Code

Customer Logo File Name

Enter the file name of the customer's logo that you want to appear on the web page when the data tag ?tf?cust_logo? is used. If a customer's logo image is not available, TopForm recommends that you use an image file that says "Your Logo Here". If this field is left blank, then you should not use the ?tf?cust_logo? tag on your Web.ec web pages.

Enter the Customer image file name

Example: cust_logo.jpg

Company Logo File Name

Enter the file name of your company's logo that you want to appear on the customer's web page when the data tag ?tf?company_logo? or ?tf?company_logo_image_source? data tags are used. To use this field, your company logo image file must be stored in the location defined in the "Customer Image Path" field below. If this field is left blank, then the company logo file name is pulled from setting in the Web.ec System Parameters.

Enter your company logo image file name

Example: comp_logo.jpg

Customer Image Path

Enter the path to the directory where the customer's logo file and your company logo file are  stored in the WebEC system. Each Web.ec customer must have their own images folder. For consistency, TopForm recommends that you name each customer's images folder after the customer.

Specify the complete URL to the location of the images.

Example URL for Company "ABC": http://166.102.29.126/images/ABC/

Try Opposite Whse if not enough Inventory? (Checkbox)

This checkbox is used to designate whether or not the Web.ec system should try the alternate warehouse assigned to this customer. If checked, Web.ec will automatically check the alternate supplying warehouse. If unchecked, Web.ec will only pull stock from the default assigned warehouse. This option is disabled if a Warehouse Group has been assigned in the field above.

Activate Monthly Order Maximum? (Checkbox)

This checkbox allows you to select whether Web.ec will apply order maximum limits to all buyers associated with this customer (see Order Maximum Limits for more information). If checked, order maximum limits will be applied for all buyer’s associated with this customer. If unchecked, order maximum limits will be disabled for all buyer’s associated with this customer

Order Maximums: Accumulate max. over (months)

Enter the number of months (1-12) for Web.ec to evaluate customer order maximums. See Order Maximum Limits for more information.

1-12 (Integer)

Activate Ship-to Restricted Ship Via List? (Checkbox)

This checkbox allows you to restrict the ship-via drop down list for all buyers associated with this customer. If checked, the ship-via drop down list for all buyers associated with this customer will contain only the default ship-via method that is associated with the selected TopForm ship-to record. This feature requires that the default ship-via method be set for all ship-to’s of the Web.ec customer. If unchecked, the ship-via drop down list is not restricted.

Checkbox

Checked or Unchecked
