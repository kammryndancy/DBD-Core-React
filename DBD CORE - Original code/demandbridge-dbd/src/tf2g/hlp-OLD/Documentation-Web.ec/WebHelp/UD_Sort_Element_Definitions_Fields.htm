UD Sort Element Definitions dialog fields and settings

The following table contains the field descriptions and the setting options and requirements of the UD Sort Element Definitions dialog.

Field Name

Purpose / Use

Setting Options and Requirements

Customer Code

Enter/Select the Customer code.  

Valid Web.ec Customer code

Sort Code

Enter/Select the Sort Code you want to define.

 

Element

Use this field to enter the sequential number you want to assign to the Sort Code element. For example, if you want the Customer Code element to appear first, enter "1" in this field, and select the "Customer Code" setting in the "Data Position" field in this dialog.

Integer

Data Type

Enter the type of data for this element. The following settings are available:

  • 1 = Data

  • T = Text

Text elements are only used for the definition of drop down lists where you may want to insert commas or other punctuation for display purposes.

1 / T

Source File

Select the appropriate Source File that contains the correct data. This field is only used for data types. A list of the available files is displayed at the bottom of the screen.

See Dialog Fields and Settings for the UD Sort Codes item.

Data Position

This field is only used for data types. Use the button to search and select the data element to be used.

Existing Data Element

Desc/Text

This field is only entered for text types. If you are defining a text element, enter the text to be used. Otherwise leave this field unchanged (data types will auto-fill this field.

User-Determined

Length

Enter the character length of the text entry and press the Enter key.  If you are defining a data element then this field should remain unchanged.

User-Determined

Data Type

All data types should currently be set to “S”. By adding “R” after the “S” (the entry in data type would be “SR”), the element you are defining will be sorted in reversed or descending order. This is useful when sorting dates in that they will appear most recent entries first.

S or SR

Account Type

All account type entries should currently be left blank. If any value has defaulted into this field, including those for data types, blank them out using the spacebar.

Not used
