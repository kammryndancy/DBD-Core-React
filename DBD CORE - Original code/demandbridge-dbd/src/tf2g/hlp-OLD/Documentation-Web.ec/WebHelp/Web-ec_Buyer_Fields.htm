EC Buyer Setup dialog fields and settings

The EC Buyer Setup dialog is used to specify the system-wide operating parameters for a Web.ec Buyer. The EC Buyer Setup dialog is divided into seven tabs. Click on any of the links below to see the field descriptions and the setting options and requirements for each tab. Click on the "Back To Top" link associated with each table to return to these links.

Field Name

Purpose / Use

Setting Options and Requirements

Buyer's ID Code

Enter the login code that the Buyer will use when logging onto your Web.ec site. See User Instructions for instructions to set up a new Buyer ID or change an existing one.

User-Determined

Buyer Name

Enter the Web.ec Buyer's name. This field can be displayed on the site's web page templates using the ?tf?buyer_name? data tag.

User-Determined

Buyer's Logon/Password

Enter the Buyer's password that the buyer must use to enter to gain access to Web.ec. Note the following password considerations:

  • The password field is case sensitive, but can be a combination of upper case and lower case letters and numbers.

  • The password cannot contain spaces.

  • Once the password is entered and the cursor is moved off this field, the password will disappear until the cursor is moved into this field again.

User-Determined

Buyer's Logon/WebEC Administrator? (Checkbox)

If checked, this field allows for the buyer to be used in a remote query of the TopForm Web.ec Server status. This operator would normally be the TopForm Web.ec Administrator. Leave unchecked for all other buyers.

Checkbox

Checked or Unchecked

Buyer's Logon/Testing?  (Checkbox)

This field is used when training buyers and employees how to use Web.ec. If checked, all Web.ec functions are available but the system will not generate a live order (or requisition). A transaction will be submitted but it will return a sample release number of 9999zzzz on the Order Confirmation page. If unchecked, all Web.ec functions are activated, including the generation of live releases and printing of delivery tickets.

Checkbox

Checked or Unchecked

Buyer's Logon/Session Expires (minutes)

Enter the number of minutes a Buyer's login should remain open without any activity. TopForm recommends that you set a limit as a safety measure for both you and the Web.ec Customer. Note: If a session expires due to inactivity and an order was in progress, the information will still be accessible if the buyer logs back in using the relogin.htm page and clicks the "Review Order" button. However, if the Buyer closes the browser or returns to the main logon.htm page, the information that was left unattended will be lost.

Integer (0-999)

Buyer's Company and Location/Customer Code

Enter the Customer code to which this Buyer belongs. If the Buyer has no Customer code in TopForm, you must go through the steps necessary to set up a customer in TopForm and then return to this step. Customer codes are set using the TopForm Accounts Receivable module.

Valid TopForm Customer code

Buyer's Company and Location/Location Code

This field can be set for either location based buyers or corporate purchasing agents. For location-based buyers, enter the ship-to code for this Buyer. All orders placed will automatically be set only to this location and no manual overrides will be allowed. If you do not know the Ship-to code, select the button to see a current list of ship-to codes. Since you have specified a ship-to location to which this buyer is tied you will also have the options to restrict order status and catalog item display specific to this location. These options are described in the “Restrict order status” and “Only show active item” options described later. For corporate purchasing agents, leave this field blank. This will enable the purchasing agent to specify a ship-to location from a drop-down menu.

 

Buyer Ship-to code or leave blank

Buyer's Company and Location/Template Code

This field allows you to specify a prefix to be used for web pages for this Buyer, enabling you to set up special pages for the Buyer if needed. For example, if you do not have or want to display item images for this Buyer's catalog, you could set up a product catalog detail page called "noimageprddetail.htm” (assuming your product catalog detail page was prddetail.htm). By putting “noimage” in this field, the TopForm Web.ec server will first search for the “noimage” prefix when a request is placed by this Buyer for the product catalog detail. If the server cannot locate web pages prefixed with this entry, then the standard web page templates will be used.

User-Determined

Buyer's Company and Location/Supply Whse

This field identifies the supply warehouse for the location(s) from which inventory will be selected for warehouse releases. This field is view-only and cannot be modified. Its source is as follows:

  • E-Logistics Module: If you are licensed and using the E-Logistics Module, the system will display the warehouse group set that is in effect for this buyer.

  • Standard WebEC: If you are not licensed and using the E-Logistics Module, the system will display the warehouse default or the warehouse setting for the customer order ship-to record associated with this buyer.

Location Base Buyers

The system will first look to the ship-to location identified in this setup to see if a default supplying warehouse has been set up. If no warehouse is set up at the ship to level, then the default supplying warehouse will be taken from the TopForm Forms Management System parameters for this customer.

Corporate Purchasing Agents

The default supplying warehouse will be taken from the TopForm Forms Management System parameters for this customer.

Values Displayed

Need to Set: In this situation, no warehouse default has been set. Order entry or Inventory Reporting cannot be done until a default warehouse is set.

Cust:xxxx: This is showing that the default warehouse is being taken from the customer’s record Note:  If the buyer places an order for a ship-to location, and the ship-to location in TopForm A/R has a default supplying warehouse, then the default supplying warehouse will be used when generating the requisition.

Ship to: xxxx: This is showing that the default warehouse is being taken from the ship to location entered earlier.

 

Buyer's Company and Location/TopForm Company

Enter the TopForm company code assigned to the company providing the Web.ec service

Valid TopForm company code

Buyer's Company and Location/Buyer Type

Enter the code used to categorize the Buyer for code-based multiple catalog assignment. This code is created using the Buyer Type Setup item from the System Setup/Supporting File Setup menu.

User-Determined

The fields in this tab control which types of TopForm orders will be displayed on the order status summary screen.

Field Name

Purpose / Use

Setting Options and Requirements

Sales Orders (Checkbox)

If checked, this field enables Web.ec Buyers to retrieve the status of sales orders (This feature is not currently available for Order Entry but is available for Order Status). If left unchecked, no information on Sales Order status will display.

Checkbox

Checked or Unchecked

R-Type Requisitions (Checkbox)

If checked, this field enables Web.ec Buyers to enter and check the status of Requisitions. If left unchecked, no information on Requisitions status will display.

Checkbox

Checked or Unchecked

W-Type Requisitions (Checkbox)

If checked, this field enables Web.ec Buyers to enter and check the status of Billable warehouse releases. If left unchecked, no information on warehouse release status will display.

Checkbox

Checked or Unchecked

Standard Sort

Select the user-defined sort code from the Customer parameter settings to use as the default sequence when the Buyer selects an order status link on the Web.ec site. You can only assign a sort code that has been set up on the corresponding customer's parameters. You can choose between Customer based sorts (H1, H2 or H3) or location-based sorts (L1, L2 or L3) from the customer parameters.

H1 / H2 / H3

L1 / L2 / L3

Restrict to Ship-to Location? (Checkbox)

This field allows you to control the display of order status information for this buyer, by location. If checked, Web.ec will only display orders designated for delivery to the buyer’s ship-to location. If unchecked, Web.ec will display all orders for this customer.

Checkbox

Checked or Unchecked

Ship-to DDL Group? (Checkbox)

This field allows you to control the display of order status information for this buyer, by group. If checked, Web.ec will only display orders designated for delivery to the buyer’s ship-to DDL Group. If unchecked, Web.ec will display all orders for this customer.

Checkbox

Checked or Unchecked

Restrict to Active Items?

This option is only available if a ship-to code has been entered in the Buyer's location code. If that field is left blank then this field will be bypassed.

Setting Comments

Y  = Will only show forms in the catalog that have been shipped to the ship-to code (as identified in the Buyer's location code) at least one time. Identification of these forms is based on the customer's ship-to location in the Custom Item (F/M - S1 Screen 8).

N  = Will have no effect.

Access

Each catalog category can be assigned a two character secure code. In order for a buyer to have that category displayed, they must have a matching two character secure Item code in this field. A maximum of ten, two character secured Item codes may be entered in this field.

Match secure Item code

6. Order Entry tab

Back To Top

Field Name

Purpose / Use

Setting Options and Requirements

Allow Order Entry?

This option controls whether WebEC will allow items to be ordered on-line. This field should always be checked for active Web.ec service. Note: Access to the order entry page is NOT controlled by this feature, but could be controlled by an available path to the page.

Setting Comments

Checked: Buyer can enter orders for items in their product catalog

Unchecked: If buyer tries to enter orders for items, all requests will be ignored preventing them from adding items to their shopping cart.

Checkbox

Checked recommended

Customer P/O

Any entry made in the Default Customer P.O. field will automatically be placed on orders placed on the web if the “buy” of the order is done and the customer P/O field has been left blank. This field will allow up to 15 characters.

Note: If the buyer does not normally use PO numbers you should at least enter the buyer id in this field.

User-Determined

Attention

Enter the permanent default for the attention field. This field corresponds to the order comments field on the order entry web page templates. If these fields contain values on the Buyer’s record, then when an order is initiated on the web, the values will be used as the defaults for the corresponding fields on your web page template.

User-Determined

Comment

Enter the permanent default for the comment field. This field corresponds to the order comments field on the order entry web page templates. If these fields contain values on the Buyer’s record, then when an order is initiated on the web, the values will be used as the defaults for the corresponding fields on your web page template.

User-Determined

Ignore Order Maximums?

This field is only accessible if the parameters for the customer associated with the buyer have had the order maximum limits activated. This is done on the customer parameters setup.

Checkbox

See Order Maximum Limits

Perc. of Avg. Monthly Usage

This field allows you to specify a maximum ordering level based as a percentage of the average monthly usage for all products being ordered. This maximum level check is only applied if location based quantity levels (in eaches) are found to not be set up on either the location record for the product (TopForm O/P) or on the Webec Item Parameters Setup.

See Order Maximum Limits
