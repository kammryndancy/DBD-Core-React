UD Sort Codes dialog fields and settings

The following table contains the field descriptions and the setting options and requirements of the User Defined Sort Code Maintenance dialog.

 

IMPORTANT NOTE: TopForm pre-defines a number of sort codes for your use (DDL_SHIPTO, EMAILSUB_O, OS_BYDATE, etc.). You can use the UD Sort Element Definition item to define the data and text elements used in these sort codes, but do not modify these codes.

Purpose / Use

Setting Options and Requirements

Sort Code

Enter a new or existing Sort Code, or use the button to view the list of existing codes. The codes for "Drop down ship-to list" (DDL_SHIPTO) and "Customer specific ship-via lists" (SHIP_VIA), and "Email Subject Lines" (EMAILSUB_O) are reserved and should not be modified, except for using the UD Sort Element Definition. The Sort Code name for the remaining types must conform to the following naming conventions:

  • SHIP_VIAxx: Buyer specific ship-via lists, where xx is two character code that makes the list buyer-specifc.

  • OSxxxxxxxx: Order Status Sorts. These codes must begin with "OS", and can contain no more than ten (10) characters

  • SHIP_VIAxx

  • OSxxxxxxxx

Description

Enter a text description for the Sort Code based on the code's purpose. For order status sorts, this description will be used as the link reference on the web pages. Therefore, you may want to use “your PO#” or “by Order Date” as for these descriptions.

  • EC3SOR: Order status sort

  • <blank>: Customer specific code definition (i.e., Ship-via)

Type

Select the appropriate type for the Sort Code.

  • D = Drop down list

  • O = Order Status

  • C = Customer specific code definition

  • E = E-mail subject line

Cust P/O Sort?

This Checkbox is only used if the "Type" field is “O”. Check this box if the order status sort you are defining is based on the customer P/O number. If your web pages support the direct access of order status information by customer P/O, then you must defined an order status sort by customer P/O and assign it to ALL customers.

  • Customer Specific Code (Type: C): FM0 (zero) (Customer Ship-to File)
