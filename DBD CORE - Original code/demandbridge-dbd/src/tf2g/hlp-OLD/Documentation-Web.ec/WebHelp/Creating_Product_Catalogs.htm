Creating a Product Catalog

You can manually create catalogs using the Catalog ID Maintenance dialog. You can access this dialog as follows:

  1. From either the Catalog Item Setup or the Product Catalog Load Utility click the   button next to the "Catalog ID" field.

  2. In the Catalog "lookup" dialog that appears (Catalogs), click on the button.

  3. The Catalog ID Maintenance dialog appears with all fields empty. The figure below shows a filled dialog.

The following section describes the use of this dialog.

Catalog ID Maintenance dialog fields and settings

The following table contains the field descriptions and the setting options and requirements of the Catalog ID Maintenance dialog.

Field Name

Purpose / Use

Setting Options and Requirements

Catalog ID

The key to the catalog used to identify the catalog internally for lookup functions..

User-Determined

Catalog Type

Custom: This catalog type will support either custom items as well as stock items. All automated catalog loading functions are directed to the loading of custom items.

Stock: This catalog type will only support stock items. Flagging the catalog in this way provides for the ability to provide special handling for this type of catalog in the future. Currently, catalogs flag with an “S” will only support the automated loading of stock items using the item selection code definitions.

Note: Defining a stock catalog will restrict the loading parameters accordingly. Only those functions with <STOCK> in the description of their functions are available for Stock catalog loading.

 

Custom or Stock

DDL Description

This is the on-line description of the catalog that will be used in all catalog drop-down lists.

User-Determined

Restrict Reports

If checked, Web.ec will prevent the catalog from appearing in catalog drop-down lists use on Web.ec on-line reports. This is mainly designed for stock catalogs from which you may wish to restrict user reporting.
