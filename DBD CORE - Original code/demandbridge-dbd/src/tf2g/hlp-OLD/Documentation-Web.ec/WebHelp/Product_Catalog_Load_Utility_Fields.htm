Catalog Load Utility dialog fields and settings

Clicking the "Load Utility" item from the Product Catalog menu launches the Catalog Load Utility dialog. The Catalog Load Utility dialog is divided into four tabs. Click on any of the links below to see the field descriptions and the setting options and requirements for each tab. Click on the "Back To Top" link associated with each table to return to these links.

Category Source

Select the source to be used for the catalog categories from the options listed below:

  • P = TopForm Product Code (use for Initial Catalog Load)

  • F = Function field on the Custom Item (see note below)

  • R = Reorder Group Code (Custom Item)

  • E = Use the default categories set on the Web.ec Custom Item Parameters

Note: The Function Code in the Custom Item (Forms Management) is a 12 character field that can be used as an auxiliary field for identifying forms.  It can also be used for BERT or ODBC functions, but it cannot be used for look-ups.

Catalog Item Code Source

Enter the code to be used to identify forms/items on the web product catalog. Select from the options listed below:

  • F = TopForm Custom Item Code

  • C = TopForm CIC Code (Customer Item Code)

  • G = Function field on the Custom Item Master

Web catalogs are sequenced within a category based on the catalog item code that is used. Using the function field as the source for the catalog item allows for you to assign a sequence to the items independent of either the custom item code or the cic code. When being used for this purpose, you will want to use the data tag ?tf?item_code? as the reference for the item code or ?tf?cic_code? as the reference for the cic code.

F / C / G

Clearing Controls

Select the clearing option you want according to the following:

  • A = Delete any existing items in this catalog prior to loading the new items.

  • B = Delete any of the Web.ec Custom Item Parameters for this customer prior to loading the new items.

  • C = Do both A & B above.

  • N = Do not delete any existing items or Web.ec Custom Item parameters in this catalog.   

A / B / C / N

Clear After Update?

Check this checkbox according to the following considerations:

  • Checked: Clear this catalog loading parameters record for this customer after this update is finished.Note: This does not refer to the clearing of the catalog or any of the customer information.

  • Unchecked: Do not clear this record for this customer after this update is finished.

Example: If the majority of your customer's catalogs are set-up the same way and repeatedly use 95% of these catalog settings, then you would not check this box so you would retain the settings the next time you returned to this screen.

Only Load If/CIC Code Set?

If checked, Web.ec will only load forms that have something in the CIC Code field.  This is primarily set if the catalog item code source is the CIC code, as you would only want to load those forms for which a CIC code had been assigned.

Checkbox

Only Load If/FMS Forms?

If checked, Web.ec will only load forms if the FMS Code in the TopForm Forms Management System is activated.

Checkbox

Only Load If/FMS Price >0?

If checked, Web.ec will only load forms if the FMS price in TopForm Forms Management System is not zero.

Checkbox

Only Load If/Function Field Set?

If checked, Web.ec will only load forms if the function field in the Custom Item in TopForm Forms Management System is not blank.

Note: This field will be automatically checked if option “G” is being used for the catalog item code source. The function field is a twelve (12) character field on the first screen of the Custom Item. If you are using entries in this field to flag products to be loaded into the WebEC catalog then you will want to say “Y” at this entry.

Checkbox

Function Field Value (1-4 fields)

If you have specified to only load custom items whose function field is loaded, you will then have the option to specify which specific function field values are to be loaded. You may specified up to four (4) values to load. If all of the value fields are left blank then the utility will revert to loading any custom item that has a non-blank function field.

Checkbox

Load Inventory Item Type

Select the option according to your requirements:

  • A = Load any of the Custom Items that have been inventoried in any of the warehouses at any time in the past.  

  • B = Only load Custom Items that have current inventory balances greater than zero in any of the warehouses.  

  • N = Load all of the customer's Custom Items. Will ignore this option.

Overwrite Existing Entries?

Choose your option according to the following:

  • Checked: overwrite any existing items image name references storred on the Custom Item parameters (E/C-S3) as well as existing catalog entries (E/C-C2)

  • Unchecked: Will not overwrite any existing items or WebEC Custom Item parameters in this catalog.

Checkbox

Image Format

This field field controls the loading of Item image file names on the Item Parameters Setup. Custom Item image file name prefix include the following:

  • C Raw customer code (without the dash 1012345670) & Custom Item Code

  • D Formatted customer code (with the dash 10-1234567) & Custom Item Code

  • F Custom Item Code (1234)

  • N No loading of Custom Item images

Note:  If the customer's forms were scanned and named using only the Custom Item code (1234), the Item image format field is used to reference and access the images that are stored on the WebEC server.  For example:  F = Custom Item code above will find images which are named using the Custom Item code.  

Special Characters

Special characters are automatically stripped from the image name when generated. The special characters filtered are set up on the second screen of the Web.ec System Parameters. Embedded blanks are stripped out automatically.
