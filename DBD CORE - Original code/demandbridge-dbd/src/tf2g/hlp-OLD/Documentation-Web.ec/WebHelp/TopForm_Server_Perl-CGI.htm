TopForm Server Perl-CGI Script Administration

All of the following administration tasks relate to the ecserver.pl and  ecserver_dev.pl scripts maintained on your TopForm Server. See Web.ec system components and Web.ec transaction processing for more information on the role of these scripts in the Web.ec system.

Perl Server Script Administration

On your TopForm Server you must start the ecserver.pl and ecserver_dev.pl scripts. If they are not running, no requests from your web server will be processed.

Checking Perl Scripts

On your TopForm UNIX server, the following command lines will allow you to check if the script is running

Removing Web Request Files

Before restarting the Perl scripts and TopForm web server you may want to remove all of the old web request files to obtain a clean start. These files contain the request being sent down from the web server. If one of the request files is causing the server to hang then it will need to be removed or the server will just hang on the same request.

 

WARNING: Removal of files in UNIX is permanent and cannot be recovered, and removal of file using the UNIX root login can cause permanent operating system damage if not executed correctly. Do not proceed if you are not familiar with the following command set. ALWAYS MAKE SURE YOU ARE IN THE CORRECT DIRECTORY BEFORE EXECUTING A REMOVE!

 

Note: Information in these files may contain order information that has been placed by an on-line user or secondary web application that you may be using. Removal of these files will lose that information. If you have concerns in this area, please contact a TopForm support representative before removing these files.

Starting Perl Scripts

On your TopForm UNIX server, the following command lines will execute the script.

 

Step

Action

Results / Comments

1

Login as root

 

2

Type in

 

#  ksh

This changes your Unix shell to the Korne shell. The Korne shell allows background processes to continue executing after you have logged off.

3

Type in

 

SCO Systems

#  /etc/rc2.d/S99ecserver

 

AIX Systems

#  /etc/S99ecserver

This is to start the live server script

 

Note: The server is configured to attempt restart automatically after a system reboot.

4

Type in

 

SCO Systems

#  /etc/rc2.d/S99ecserver_dev

 

AIX Systems

#  / etc/S99ecserver_dev

This is to start the development server script

 

Note: The server is configured to attempt restart automatically after a system reboot.
