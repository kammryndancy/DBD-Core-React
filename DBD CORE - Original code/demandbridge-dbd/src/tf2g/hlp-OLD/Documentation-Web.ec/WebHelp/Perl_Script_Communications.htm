Perl Script & Communications with the TopForm Web Server

Web.ec uses a Perl script to handle communication between web page requests and your TopForm Web.ec server. When a buyer on your web site clicks on a link or presses a button, the pages request the Perl script (cgi-bin/ecclient.pl) to process the request, send it to the TopForm Web Server and wait for a response (the response being the resulting web page with the requested information).

 

In your live Web.ec environment, the communication file is suffixed with “.ec” and is placed in the “www_interface” directory on your TopForm server. When the TopForm Web Server sees a file in the “www_interface” directory with this suffix it knows a request has come in.

Web.ec Development Environment

With your development environment, the development pages will process requests with a different Perl script (cgi-bin/ec_dev.pl) that will process files with a “.dev” suffix into a “www_dev” directory on your TopForm server. A separate TopForm Web Server will be standing by looking for files with the “.dev” suffix that appear in the “www_dev” directory.

 

See Web.ec system components and Web.ec transaction processing for more information.

Perl Connectivity

There are two methods currently in use for communication between your Web Server and your TopForm server using Perl.

 

Port Communication (Internal or External Web Server)

Primary Connectivity Method: This method is the primary method, and uses two Perl scripts, one on your Web Server and one on your TopForm server. The two scripts "talk" with each other over a specified port between the two systems.

 

The Perl scripts in this environment are as follows.

Drive Mapping (Internal NT Server Only – Samba Required)

Note: This method is only in use if you are specifically advised by a TopForm representative.

 

Port communication is the primary connectivity method in use. This method the maps a drive on your Web Server to your TopForm server “www_interface” directory. The drive mapping for your live WebEC environment must be drive “W”. In this environment, the Perl script on your Web Server post all requests to drive W which places them in the “www_interface” directory on your TopForm server.
