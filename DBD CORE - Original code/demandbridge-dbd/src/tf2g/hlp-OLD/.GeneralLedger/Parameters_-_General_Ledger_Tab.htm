Parameters - General Ledger Tab

 

G/L Account Number Format

 

Specify the Length for each segment of the G/L Account Number. The Account number may be entered and displayed in up to four segments. The total number of digits cannot exceed twelve. For example:

 

Format

Enter As

xxx-xx

3-2-0-0

xxxxx-x

5-1-0-0

xxx-xxx-xxx-xxx

3-3-3-3

 

Most companies will define a chart of accounts where the first segment relates to the type of account. The second segment would be used to define sub-accounts and if the company is divisional, the third segment would be used to identify the division.  

 

Following the definition of your general ledger account format, you should probably proceed to setting up your chart of accounts as many of the following Parameters and the Other Setup programs will require you to enter General Ledger Account Numbers.  You could of course use the setup option in each program to set up the accounts as you need them, but it is probably more efficient to get this out of the way first.

 

General Ledger Account Number Setup

 

 

 

Enter each general ledger account number and the appropriate description.  Note that at any point you can use the query button to see the accounts that have already been defined.

 

 

There are also two utilities found in the System / System Setup folder that enable you to copy general ledger account numbers.  One of these utilities allows you to copy all or a range of account numbers from another TopForm company for which you have already setup operations.   The other utility provides the ability to copy departmental accounts.

 

Copy Company Chart of Accounts

 

 

Source Company

 

Enter the three character company code whose chart of accounts you wish to copy.

 

Destination Company

 

Enter the three character company code for the company you are now setting up.

 

Remove Existing Accounts

 

If there is a chart of accounts already in existence for the new company, checking this box will replace it in its entirety with the new chart.

 

Overwrite Existing Accounts

 

If there is a chart of accounts already in existence for the new company, checking this box will replace only the account numbers that would otherwise be duplicated by the new entrees. Other account number will remain intact.

 

Starting and Ending Account Numbers

 

Enter the range of account numbers you wish to copy.

 

Copy Departmental Chart of Accounts

 

 

This utility allows you to set up your chart of accounts for one of your branch offices or operating divisions and then copy the account numbers to create as many other branches as you need.  You can also use this utility to update the accounts in your divisions as you make changes.

 

Source Department

 

Enter the source department or branch where you have set up the prototype account numbers.

 

Destination Department

 

Enter the first new department or branch that you are creating.

 

Replace Existing Account Descriptions

 

When using this utility to update account numbers that you have changed, you can check this box to replace the existing account descriptions.

 

Copy Balance Sheet Accounts

 

Check this box to also copy Balance Sheet accounts.

 

Copy Budget Information

 

Check this box to duplicate Budget Data

 

Parameters – General Ledger

 

 

After setting up your chart of accounts, you are ready to complete the General Ledger parameters.

 

Prior Years Adjustments Account

 

The system provides for reopening a closed fiscal year. Any posting made to a closed fiscal year must be brought forward to the current fiscal year. If you make a prior year adjustment to an Income Statement Account, it must again be closed out to an account in the Equity Section. TopForm® Accounting does not assume you want this amount to be posted to Retained Earnings, and therefore, gives you the flexibility to specify an alternate account number to be posted.  Enter this account in the “Post prior year P/L Adjustments in current fiscal year to account” parameter.

 

Periods To Retain G/L Postings

 

Any number of periods of general ledger postings may be retained.  As disk space has become so inexpensive, set this number to the maximum that you would ever want to report or display.  We suggest from 36 to 60, however it can be set as high as 99.  

 

Account Balance History

 

Each General Ledger Account has an Account Balance record containing the summary totals for each of the accounting periods in the fiscal year.

 

Account Balance Panel

 

 

The information in this record is used to produce your financial statements. The Financial Statement program can also print comparative statements, that is, this year’s income versus the same period last year. Don't worry, if you are just going onto the system, you are not required to load this information. The system will automatically save it at year-end, giving you comparative statements starting next year. Of course, you may enter last year's information to immediately utilize this feature. To take advantage of comparative financial information, enter at least “1” for the number of prior years to retain Account Balance history. To retain more than just last year's history, enter the number of prior years history you wish to maintain instead.  Once the information has been purged as a result of year-end processing, it cannot be retrieved (except from backup), therefore we recommend that you set this to three to five years.

 

Current Fiscal Year

 

Enter the Calendar Year in which the Current Fiscal Year ends. For example, if today is November 30, 2004 and the company is on a Calendar Year, enter 2004. If you were on a Fiscal Year ending June 30, enter 2005 instead. The Current Fiscal Year is used as a default in most programs.

Accounting Method

 

Enter the appropriate code for Which Accounting Method should be used.  Most companies will chose “Accrual.”   Later in the set up process, Cash Basis companies will be given the additional option of also reporting on a accrual basis for managerial purposes.  Please note that this does add some operational complexity to your accounting system.  Do not change this parameter after beginning processing

 

Next Journal Entry Number To Assign

 

TopForm provides for the system to automatically assign sequential journal entry numbers . If you have been using journal entry numbers, enter the next available number. If you have not previously used journal entry numbers, you may begin with any number, for instance, the number 000001. It is not necessary to imbed the date or the period into your numbering scheme as all transactions will have a date and an associated accounting period attached to them for inquiry.

 

Last Fiscal Year In Which Recurring JE’s Were Updated

 

This parameter helps you keep track of your Recurring Journal Entries so that you do not update them more than once within the fiscal period. Enter the period preceding when your next Recurring Journal Entry will need to be updated.
