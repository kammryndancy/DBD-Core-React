Requisition Charge Options

 

Below is a listing of fields from the Requisition Charge Setup screen and a description of the field.

 

Field Description
Price

If desired, enter a price for this charge. Note: based on what you select in the "Calculation Method" field will determine if the number entered in this field is treated as a fixed amount or a percentage of cost, sales, or value. The same is true for the "Cost or SLSP GP Adj" field.

P-Minimum

P-Maximum

 If desired, enter a minimum and/or maximum extended price in these fields.
Cost or SLSP GP Adj If desired, enter an amount to adjust the Cost or Salesperson GP. Enter as a negative to reduce salesperson GP.
Comm Code Enter a commission code if you would like to set a specific commission code to be used for this charge (for "line" type commissions only).
Calculation Method Choose if the Price and/or Cost or Salesperson GP adjustment field should be calculated as a percent of sales, percent of cost, percent of value, a fixed amount, or if a table should be used.
Type of Charge Choose if this charge should be per line, per requisition, or per carton.
Apply to Order Type

Choose if the charge should apply to standard invoices, invoices for releases, invoices for billable releases, or more than one invoice type. (Enter any combination of I, R, and W in CHUI, or hold the ctrl key to select multiple types in GUI.)
Apply to Terms Code Enter a terms code if this charge should only apply to invoices with a specific terms code. Note: Use of the terms code field overrides all other options
Should this line be an Acct Cost? Mark if the line should be an Accounting Cost. "Yes" will generate a cost only accounting line, "No" will treat this as a salesperson GP adjustment.
Memo Charge This field is currently not used.
Don't print on invoice if Zero amount? Mark if the line should not print on the invoice if it is calculated as a zero amount.
Omit from B/O's? Setting this to "yes" means the charge will not be applied if order/req has already been billed.
Apply to supply items? Mark if this should apply to supply items.
Apply to ship via codes If desired, enter ship via codes that this charge should apply to, up to 5 ship via codes can be entered.
Warehouse Code - Out Enter a warehouse code if this only applies to specific outgoing warehouse.
Warehouse Code - In Enter a warehouse code if this only applies to a specific warehouse an item is coming into.
Inventory Ownership If desired, designate if this charge should apply only to distributor owned or customer owned goods.
Apply to P/C Set If desired, enter a product code set that this charge should apply to. If any line on the order contains any of the product codes that are in the Code Set, the automatic requisition charge will be created.
Apply Min/Max to GP Adj. or Cost? The P-Minimum and P-Maximum fields were originally designed to work if your charge was set as a "per requisition" charge. An enhancement was made that allows the min or max to be applied to "per line" charges as well. You can select if you would like the min/max to apply to the price field, GP Adj., or Cost only line. To not use this enhancement, choose "none".
Only apply to BO Fill Report orders?

On the Backorder Fill Report there is an option to flag customers for Auto Req Charges. That question is used with this question to determine what orders should be charged.

You can answer Only Flagged Orders (Y), All Orders (N), or Only Flagged orders AND Only this Req Charge (O). If the user answers Y, then only orders flagged when the Backorder Fill Report was run will have the Automatic Requisition Charges applied to them. If the user answers N, the Automatic Requisitions will be applied to all orders, whether they appeared on the Back Order Fill Report or not. If the user answers O, this Automatic Requisition Charge will be the ONLY one applied and it will only apply to orders printed on the Back Order Fill Report. In this last case, even if other Automatic Requisition Charges also meet the criteria for appearing on an invoice, they will NOT be applied. Only the one Automatic Requisition Charge with the O for only flag will be applied.

Cost Code Data should not be entered into this field at this time.
Prompt before adding charge to invoice? You can select if you would like to receive a message before the requisition charge is added to the invoice.
