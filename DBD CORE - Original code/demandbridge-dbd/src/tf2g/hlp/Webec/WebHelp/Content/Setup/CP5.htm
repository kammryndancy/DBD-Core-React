Customer Parameters - Tab 5 - Controls

 

Use the following controls to determine the eCommerce system behavior with the customer:

Field Description
Activate order maximums Checking this on will activate the order maximum limits for buyers associated with this customer.
Activate ship-to restricted ship via list Checking this on will restrict the "ship via" drop down list for buyers associated with this customer to only show the default ship via method associated with the customer's ship-to record. This requires the default ship via method be set for all ship-to's for the customer.
Activate single instance on item controls Checking this on will make it so that an item can only be ordered once per order. This is used to keep the user from accidentally ordering an item twice. An override can be set on the item level to allow specific items to be ordered multiple times on an order.
Alternate terms code for non-cc purchase For buyers not using a credit card for purchases, this terms code will be applied to their orders.
Accumulate max check of XX months If order maximums are activated, you can specify that the maximum can be calculated from 01 to 12 months. For example, a monthly maximum of 500, could really be viewed as a maximum of 3,000 over a period of 6 months. If left blank, or set to 0 or 1, then the maximums will be monthly.
Prevent on-hold orders from cc capture If credit card processing should be used to only capture credit card data, then set this to "yes".
Suppress use of IMS price in catalog/cart If the customer uses IMS pricing, but you do not want to show the IMS prices online, then set this to "yes".
Suppress buyer default PO until purchase Check this on if the buyer's default PO should only be used if they have not entered a new PO at the point eCommerce is creating the order.
Generate reorder notice if "R" type msg is displayed If set to "yes", a reorder notice e-mail will be sent to those parties with an "R" set in their e-mail actions when an item being ordered reaches it reorder point.
Don't show price for buyer Whatever is set here will be the default setting when setting up new buyers for this customer. Check this on if the buyers should not be able to see the price.
Allow manual ship-to address Setting this to "yes" will allow for the override of the shipping address. However, the ship-to code will remain the same.
Display EC Promo link If this is set to "yes", the buyers will be able to order items via the EC Promo module. This requires that you have purchased the optional module EC Promo.
Always use IMS price for tag price on PC If set to "yes", the product catalog will always show the IMS price, even if it is set to zero.
Freight mark-up Enter in a freight markup code if you would like for web orders to use a specific markup method.
ECP template buyer Select a buyer to be used as the EC Promo template buyer. When new buyers are setup that have access to EC Promo, they will automatically have access to the same catalogs and categories that are setup for this "template" buyer.
Color scheme If you would like for buyers associated with this customer to use a color scheme besides the default, enter the color scheme code in here.
Display SAGE link

If this is set to "yes", the buyers will have access to order items using SAGE. This requires that you have purchased the optional module SAGE Promo.
