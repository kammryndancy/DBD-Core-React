Vendor Setup Fields

 

Field Description
Web Address If available, enter in the vendor's web address.
Standard Comment If needed, enter a standard comment. This will default when entering a vendor invoice, and will subsequently print on the check when the invoice is paid. This field is normally left blank, as any needed comments would usually be entered at the time of invoice entry.
Std GL Account If needed, this will set the default GL account that will be written to when an invoice is entered for this vendor. This would typically be used with vendors where direct AP invoice entry is used, as opposed to invoices created from a PO.
Std Amount If needed, enter a standard amount. This will set the invoice amount to a specified amount. This is useful if you have a vendor the regularly bills you for a set amount.
Bank Code The standard bank code that would be used for the check printing with this vendor.
Our Account Number Your account number with the vendor.
SPC This field stores a specific program name that will be called to electronically send orders to this vendor. It is used today for the interface with United Stationers for the electronic purchase of office products.
Intercompany GL If you are using the intercompany vendor feature, entering in a GL account here will let the system know that this vendor is an intercompany vendor, and will cause all invoice amounts to post to this account, instead of creating an open invoice.
XML PO Option Entering an "E" in this field while the SPC field is set to "PO2XML" will cause the "Print this PO" option to be set to "No" when a PO is created.
Comm Sub Code The commission substitution code field allows you to enter in a 1 digit number. This number will insert itself into a specific position (as defined in your AR parameters) in order to use an alternate commission code on any lines ordered from this vendor. Example, if this is set to a "5", and the commission code was 00030, then the commission code would be changed to 05030, if the AR parameter set it to be replaced in the 2nd position. This would be used to pay specific commissions by vendor, or to have a house charge associated with a particular vendor. The resulting commission code must be setup as a valid commission code in the Accounts Receivable module.
Payment Selection Code Payment Selection Codes can be used by the Auto Invoice Selection program for check printing. This can be another field used for controlling which invoices are paid when you are selecting invoices for payment.
Priority Code Payment Priority Codes can be set between A and Z, with A having the highest priority. During Auto Invoice Selection, for check printing, you can specify that only accounts above a certain priority are selected to be paid.
On Hold? Placing the vendor on hold will keep the system from printing any checks to this vendor. Even if the vendor is not on hold, individual invoices can still be placed on hold.
Mailing Label Types These fields can be used along with the Mailing Labels report, which is used for creating mailing labels.
E/F PO Job Card If this field contains a valid jobcard, the purchase order print program will assign this jobcard only to purchase orders of the vendor. Any spec templates from the product code will be appended to the job card. For example, if the standard job card is 850PO and the job card override is 850PON, a jobcard of 850POLAB will be 850PONLAB. This field is generally left blank.
Print PO? Do you want to have the PO's marked by default to print?
File Output Format

This field is not for general use.
Always Take Discount?

Always take an available discount with this vendor, even if past due?
Include Special Items in Discount

For items setup in the Invoice Entry Options screen, input the numbers of the items for which the discount would apply.
Extra Due Days

If the vendor will accept payment up to X amount of days after the due date without any penalty, you may want to put the number of days here. This will cause the Auto Invoice Selection and Cash Requirements report to

Customer Supplied PO Type

If this field is set to "Yes", when a Stock Purchase Order is entered for this vendor, at the time of PO Receiving, it will not allow a vendor invoice to be entered. This will allow you to bring customer supplied inventory onto the system without entering in a vendor invoice.
