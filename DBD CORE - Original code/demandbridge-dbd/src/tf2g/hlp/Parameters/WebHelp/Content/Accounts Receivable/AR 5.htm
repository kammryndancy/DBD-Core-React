Accounts Receivable Parameters - Screen 5

 

DB Distributor > Accounts Receivable > Setup > System > System Setup > Parameters (ARS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Activate alternate customer code field?

When converting from another software application, you may decide to renumber your customers. If so, you may wish to store the previous customer code. By checking this box, an additional field on the address panel of the customer master file will be provided.

Create a commission by line item file? It is recommended that you turn this "on". You can use ODBC to access this file for more in-depth commission and sales analysis reporting.
Print override billing address on packing list and purchase order? If you allow operators to override the billing address and if you want this address to print on the Pack List and the P/O in addition to printing on the customer invoice, check this box.
Update statistics by salesperson, customer, product code? We recommend that you check this box. This provides for additional sales analysis reporting in that you can see sales analysis by Product Code in addition to data by Salesperson and Customer.
Sub the last 2 digits of sper code into the GL account number beginning in position Normally you would leave this field blank. It provides for general ledger reporting by salesperson. If you wish to have the last 2 digits of the salesperson code placed into the GL account code, enter in the position of the account code where it should be placed.
Sub "margin for profit" on some report headings? Certain reports have been modified to print the words Gross Margin as opposed to Gross Profit. If this is your preference, check this box.
Enable customer comment printing feature?

If checked, you are able to enter comments that will automatically print on the assigned forms. These comments can be accessed from your customer master file. We recommend that you take advantage of this feature. The forms this is available on are the Invoice, Purchase Order, Packing List, and Statement.
Enable question to setup contact management record for new customers? If this box is checked, upon entering a new customer, you will be asked if you would like to setup a corresponding contact record for them.
Update order booking by customer file? In earlier releases of DB Distributor, the customers’ statistics that updated were sales, gross profit and number of invoices. Since that time, DB Distributor added bookings and booked gross profit. These statistics update at the time of order entry. It is recommended that you select this option by checking this box.
Print invoices in foreign currency feature? Normally this box would not be checked. This feature was created for DB Distributor clients that sell products to other countries that use a different currency.
Allow WIP GL entries if no PO code? Matching is one of the Generally Accepted Accounting Principals. This means that you should post costs of sales in the same period in which you recognize revenue. When you receive vendor invoices, DB Distributor will post the cost to a Work in Progress general ledger account. At the time you invoice your customer, this accrual will be reversed against a posting to cost of sales. Normally you will leave this box unchecked as you would only want to make a posting to Work in progress upon receipt of a vendor invoice that was the result of a purchase order. However, some customers may choose to accrue work in progress for internally generated costs. Checking this box will provide for work in progress entries for special charges that are not included on purchase orders.
Use GL account numbers from sales tax code record? If you sell and report sales tax in various states, you may wish to have a different sales tax payable account by state. Checking this box will provide you with the option to enter a Sales Tax Payable and an Accrued Sales Tax general ledger account under each of your State Sales Tax Codes.
Use substitution when making WIP entries? You may wish to track work in progress by division or even by salesperson. Check this box to include these entries in separate general ledger accounts.
Enable commissions by salesperson and product code? If your commission calculations change based upon the product that is sold, you have two options. One is strictly to default commission codes based upon the product code and the other defaults commission codes by a combination of the Salesperson and the Product Code. This later method is triggered by checking this box. In doing this you can put in place plans that default commission by product code but that also vary from one salesperson to another.
Update all profit statistics using salesperson's adj. GP? DB Distributor provides the logic to add house or overhead charges before calculating commissions. If you want the gross profit that is reported on your various sales analysis reports to be adjusted based upon the house charges that are added, check this box.
Update statistics by customer and item code? In earlier releases of DB Distributor, sales and profit statistics updated by customer and separately by item. Checking this box provides for an additional level of detail, by customer and item.
Enable credit memo feature? DB Distributor created an entire system of tracking credit memos. Details regarding who (member of your staff) initiated the credit memo as well as the reason for the credit are tracked. There are no standard reports that are available for this data. It is intended that any reporting requirements would be accomplished using ODBC.
