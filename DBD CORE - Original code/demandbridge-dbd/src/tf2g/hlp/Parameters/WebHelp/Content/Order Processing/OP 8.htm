Order Processing Parameters - Tab 8

 

DB Distributor > Order Processing > Setup > System > System Setup > Parameters (OPS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Update unbilled cost when receiving cust owned if different?

If you initially receive inventory without the vendor bill, you might not know what the actual cost is. If this parameter is turned on, if the vendor bill is later received with a different cost, then the unbilled costs and cost on the lot records will be updated. Click here for details.
Update line cost and order booking upon receipt of vendor invoice? If setting the previous parameter to "yes", you have the ability to take this one step further, and have the new/corrected cost update to the order line, for historical purposes, and to have the order booking report update with the new cost to have an accurate "booked profit" calculation. Click here for more details.
Implement "freight handling rules by order" feature? If this is set to "yes", then a new field will be displayed in the order entry header screen that will contain the freight markup code to be applied to the order. This will contain the default from the customer, but can be overridden.
Create history for special charges added during invoice entry? It is recommended that this be set to "yes" in order to keep all order lines for historical purposes.
Default the "date requested" field to "W" and "R" orders to X? Enter F for the following business day
Enter S for the same business day
Enter N for Not applicable
Use current commission codes when creating an order with a repeat? This would normally be set to "yes". If set to "no", then the commission code on the original order will be used. This could result in the wrong commission code being used if the salesperson has moved to a new rate.
Not Implemented SSP 157380

Ignore this option at this time.
Allow entry of custom item contract expiration date for stock items? If you have custom items that you have marked as "stock" items, do you need the ability to enter contract expiration dates for these items?
Add line number option to reprint packing list prompt in OE? When reprinting packing lists from order entry, do you require that there be an option to be able to specify by line which items should be reprinted?
Update FMZ with FMS price from lot record? If set to "yes", the FMS price stored on the lot record will update to both the FMZ and FMY files when the lot is sold/shipped. The FMS price can be entered on the lot during PO Receiving or during an Inventory Transaction. If there is no FMS price on the lot, the FMS price on the item master will be used.
Use Companion Item login in manual order entry? If you are using the eCommerce module, you can specify here that you would like to use companion items in manual order entry as well.
Allow entry of RC number during Order Entry? If you are using eCommerce, and you make use of the Remote Costing feature, you can specify that this field can be entered in manual order entry as well.
Not Implemented SSP 185554 Ignore this option at this time.
Default number of days from Order Date to Ship Date? The number used here will be used to calculate a default PO ship date based upon the order date.
Not Implemented SSP 187465 Ignore this option at this time.
Use Item Class from lot record for Pricing? During Purchase Order Receiving, the receiving lot screen has been modified to allow entry of a price class. The default price class value is taken from the Inventory file.
A new program I/C-L1 has been created to allow maintenance of the lot price class for existing lot records. Only the price class can be modified in this program. Lot records cannot be added or deleted in this program.
Order entry and Requisition/Release entry have been modified to use the price class from selected lot records. If multiple lots are selected for an order line, the first lot committed to the order line will be used. Separate order lines must be created if multiple lots with different price classes are selected.
eCommerce has been modified to use price classes from lots when creating an order. Lots are not selected when a catalog page is displayed. Therefore a lot price class cannot be selected until an order is created.
Use customer terms code when selecting repeat orders? This will cause the system to use the current terms code on the customer on repeat orders, as opposed to the terms code that was used on the original order.
Not yet implemented Ignore this option at this time.
Use Output Director packing list? If this option is turned on, you have the option of sending packing list data in the XML format via the Output Director.
