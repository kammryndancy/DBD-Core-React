Accounts Payable Parameters - Screen 1

 

DB Distributor > Accounts Payable > Setup > System > System Setup > Parameters (APS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Current Fiscal Year

Enter in the current fiscal year. This will automatically update to the new year when you run the year-end process.
Current Accounting Period Enter in the current accounting period. This will automatically update to the new period when you run the period-end process.
Divisions Required? Normally this parameter is set to "N". Some companies want to separate their accounts payable into divisions or by branch office. This means that a vendor record will belong to a particular division, denoted by the first two digits of the vendor code. Furthermore, all subsequent reporting will be division oriented. This is not desirable if the same vendor could be used for each division. Electing division accounts payable results in separate vendor files for each division. If a vendor called to check on the status of a payment, you would have to check multiple vendor files. There is a method of obtaining subtotals within the Accounts Payable System by using categories (i.e. Trade, Utilities, etc.) instead of division in your accounts payable.
What size vendor code should be used? The Vendor code can be up to eight characters in length. It is recommended that you use the entire eight characters if you are using alpha vendor codes. For instance, set up Federal Express as FEDEX or United Parcel Services as UPS.
Are there multiple bank codes? If you use more than one accounts payable bank account, check the box next to "Are there multiple bank codes?" Normally there is one bank account used to print checks generated by the computer, but you could have multiple bank accounts if you have field offices that have their own local bank account.
What is the standard bank code? When printing computer checks or entering handwritten checks you will be required to enter a bank code. The system will default to the bank code you enter here. Normally, enter "A". Later, using the bank code setup program you will enter the specifics for this code.
Will the GL control accounts be interfaced by category?

The following are accounts that are posted whenever an invoice is entered or paid within DB Distributor accounting. If you check the box associated with this parameter option, you will have the ability to have different general ledger interfaces by vendor category.
What is the maximum size of the vendor category code? The vendor category code may be from one to nine digits in length. Normally a size of two is adequate. Enter the number of digits that you require.
Enter code(s) of statistical data to be retained The choices are purchases, adjustments, and disbursements. Select one or all by holding down your Ctrl key and clicking the choices. Then click the lock button. Under the Lock button you will see the letters corresponding to the static types you have selected. Purchases will be the net of adjustments, if you do not select to keep Adjustment statistics separately. If you select both, Purchases will be the gross amount of the payable invoices entered. Subsequent invoice adjustments will be kept under the separate statistical type. These statistics will only start accumulating from the date you make the selection.
Enter the corresponding number of prior years statistics to keep This field will hold your preference for the number of prior years statistics to retain. Just enter a number from zero to nine, for each of the statistics you chose to keep in the previous question. The current year will always be kept for the selected codes. Once Statistical data has been purged, you will not be able to retrieve it, except from backup. It is better to enter three or four here and adjust it down if you find that you are not using the older statistics.
Number of periods to retain vendor invoice history Enter in the number of periods of invoice history to keep. This should be set to at least 24.
Should invoice detail be printed on the Cash Disbursement Journal?

By default, the cash disbursements journal will not show the invoice detail that is being paid off by the checks. You can set this to "Y" in order to have this additional information print on the journal.
