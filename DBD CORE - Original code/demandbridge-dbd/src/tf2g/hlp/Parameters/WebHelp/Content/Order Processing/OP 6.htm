Order Processing Parameters - Tab 6

 

DB Distributor > Order Processing > Setup > System > System Setup > Parameters (OPS S1)

 

For each parameter listed below, you will see the available options, and if applicable, a recommended setting.

 

Parameter Description
Enable sales order summary display in req/release entry?

If set to "yes", this will cause a sales order summary screen to display in OP D1. CHUI only.
Enable customer's item code feature? If set to "yes", this will open up a field on the custom item master record called the Customer Item Code, or CIC for short. This can be used to hold an item code up to 20 characters long. This field can be printed on the Purchase Order and Invoice in place of the standard item code. If activated, you may also use this CIC code during the Order Entry process.
Eliminate "send letter type" for O/E header screen? If set to "yes", this will remove the field that would ask for the letter type in the order header.
Eliminate "print shipping acknowledgement" from O/E header screen? If set to "yes", this will remove the field that would ask to print shipping acknowledgements in the order header.
When out of stock in req/rel entry, warn if order exists with same ship-to? If you already have an item that is on backorder for a specific location, this feature can help you from entering a second order that would put even more of the item on backorder.
Enable receiving/billing department feature? This feature activates 2 fields that are stored on the ship-to location record, and have a length of 20 each. You will want to have this set to "yes".
Set special ship to "y" for custom forms in order entry?

If the majority of your orders for custom items are not direct ship, you can set this to "yes" to make sure this field is not overlooked.
Require a 2nd set of carton label fields by order line item? If set to "yes", the additional fields will be opened up on the order entry line.
Calculate RON suggested reorder quantity based on avg monthly usage? If suggested reorder qty is zero on the custom item master, and reorder point in days is not 0, enter an X in the box, and the number of months for calculation in the side field.
Retain entered selling price when modifying an existing order line? If set to "yes", the line will keep itself set at the previously entered price, instead of defaulting back to the item master price.
Notify operator order if "bill as shipped" in PO receiving? If set to "yes", when a Bill as Shipped order is received into PO Receiving, a message will be displayed to the operator to let them know that it is a BAS order. This will let them know that this will require them to receive inventory, and that they will not be proceeding to AR Invoice Entry.
Allow over shipment? This controls whether or not you accept overruns.
Allow exclusion of invoice on summary bill? If this is set to "yes", a field will be displayed in Invoice Entry that will allow the operator to designate if an invoice should be setup as a summary bill or not. The field will default based upon the customer settings, but can be overridden with this parameter turned on.
Enable backorder fill feature from inventory transactions? If you have an item that is on backorder, when trying to fill the backorder, do you only want to consider inventory coming in from purchase orders, or should any inventory transactions also be able to fill a backorder.
Do not bring sell unit from lot to req line for customer owned? If set to "yes", then the sell unit from the item master will always be used on the order line.
Set u/m for N-other line type in O/E as "M" rather than "EACH"? For the line type of "other", should the unit of measure be defaulted to "M" instead of "EACH"?
Automatically add freight from lots to order header freight field? If set to "yes", on release orders, any inbound freight from the lot record will be put into the header to be passed onto the customer at the time of invoicing.
Display maximum during "Location's req max exceeded" warning? If set to "yes", if an operator exceeds an item's maximum, and they receive a warning message, the quantity that is the maximum will also be displayed.
Retain message lines in order history? This should be set to "yes" in order to keep all message lines for historical purposes.
